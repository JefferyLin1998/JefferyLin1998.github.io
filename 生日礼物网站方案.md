# 老婆生日礼物网站 — 方案与实施指南

> 目标：在 **GitHub Pages**（`https://jefferylin1998.github.io/...`）上发布一个私密、有仪式感、带小互动的生日祝福站。  
> 本文档为完整方案；按文末「实施步骤」即可从零搭建。

---

## 一、整体创意：「爱的解谜之旅」

把网站设计成 **一条有剧情的闯关路线**，而不是单页堆文字。她每过一关，解锁一点回忆或一句情话，最后才看到完整情书与「生日惊喜」。

**叙事线示例：**

```
封面（密码/暗号进入）
  → 第1关：我们的第一次（选择题/填空）
  → 第2关：照片拼图 或 找不同
  → 第3关：听声音/看剪影猜地点或事件
  → 第4关：时间轴里点「最难忘的一天」
  → 终章：展开长信 + 生日祝福 + 可选语音/视频
```

**为什么这样设计：**

- 互动拉长「拆礼物」的时间，比静态贺卡更有记忆点。
- 题目内容全是你们独有的梗，她会感到「只有我们才懂」。
- 静态站即可实现，无需服务器，适合 GitHub Pages。

## 二、推荐站点结构（GitHub Pages）

在仓库根目录下新建 **`birthday/`** 文件夹，与现有密码保护配合使用。

### 方案 A：沿用当前密码保护（推荐）

你仓库里已有 [matteobrusa 式静态密码页](https://github.com/matteobrusa/Password-protection-for-static-pages)：

1. 访问 `https://jefferylin1998.github.io/`，输入你设定的密码。
2. 浏览器地址栏会出现 `#一串SHA1哈希`。
3. 在仓库根目录创建 **与哈希同名的文件夹**，把生日站放在里面，例如：

```
JefferyLin1998.github.io/
├── index.html                    # 现有密码入口（保留）
├── <你的密码哈希>/               # 例如 a1b2c3d4...
│   └── birthday/
│       ├── index.html            # 生日站首页
│       ├── css/
│       ├── js/
│       ├── assets/               # 照片、音乐（注意体积）
│       └── data/
│           └── content.json      # 题目、情话、时间轴（方便你改文案）
└── 生日礼物网站方案.md            # 本文件
```

**访问地址：** `https://jefferylin1998.github.io/<哈希>/birthday/`

- 优点：不改动现有根目录逻辑；外人很难猜到路径。
- 注意：不要把含照片的 `assets` 放在未保护目录；链接勿公开到社交媒体。

### 方案 B：不加密，仅靠「难猜 URL」

例如 `birthday-2026-secret-xxx/index.html`，只靠长随机路径保密。  
**不推荐**：容易被爬虫、目录列举扫到；仅作临时测试可用。

### 方案 C：生日当天再 push

仓库设为 **Private** 无法直接用免费 GitHub Pages；若保持 **Public**，务必用方案 A 或强密码哈希目录。

## 三、页面与互动模块（可勾选组合）

| 模块 | 说明 | 难度 | 浪漫指数 |
|------|------|------|----------|
| **封面** | 星空/花瓣动画 +「点击进入我们的那一年」 | 低 | ★★★ |
| **暗号进门** | 输入纪念日、昵称、第一次约会地点等 | 低 | ★★★★ |
| **选择题闯关** | 3～5 道「我们之间」的选择题，错题温柔提示 | 低 | ★★★★ |
| **填空题** | 「你最喜欢我叫你___」「我们结婚/在一起第___天」 | 低 | ★★★★ |
| **照片拼图** | 4～9 块拼图，完成后显示合照与一句话 | 中 | ★★★★★ |
| **记忆翻牌** | 配对游戏：相同照片翻开即配对 | 中 | ★★★★ |
| **时间轴** | 纵向滚动：相识→表白→旅行→今天，点击节点弹故事 | 低 | ★★★★★ |
| **刮刮卡/信封** | 鼠标刮开涂层或点击拆信封看信 | 低 | ★★★★ |
| **打字机情书** | 长信逐字出现，可配背景音乐开关 | 低 | ★★★★★ |
| **语音祝福** | 录 30～60 秒 mp3，点击播放（文件放 assets） | 低 | ★★★★★ |
| **倒计时/正计时** | 「我们在一起已经 ___ 天」实时跳动 | 低 | ★★★ |
| **终章烟花/蛋糕** | CSS 或 canvas 轻量烟花 +「吹蜡烛」点击 | 中 | ★★★★ |
| **小彩蛋** | 连续点击某颗心 7 次出现隐藏一句 | 低 | ★★★ |

**建议组合（约 15～25 分钟体验）：**  
封面 → 暗号 → 3 道选择题 → 拼图 1 张 → 时间轴 5～7 个节点 → 打字机情书 → 终章烟花。

## 五、视觉与氛围建议

| 维度 | 建议 |
|------|------|
| **配色** | 暖粉 `#f8e8ee` + 深玫红 `#c45c7a` + 米白文字区；或她喜欢的颜色 |
| **字体** | 标题用圆润手写感（如「霞鹜文楷」CDN），正文用系统无衬线 |
| **动效** | 轻量：淡入、飘落花瓣；避免闪屏、自动大声播放 |
| **移动端** | 必测 iPhone / 微信内置浏览器；按钮足够大、拼图支持触摸 |
| **性能** | 图片 WebP、懒加载；总资源建议 < 5MB |

**风格参考关键词：** 温柔、私密、略俏皮，不要像企业 H5 广告。

## 六、技术方案（静态站，零后端）

| 项目 | 选择 |
|------|------|
| 框架 | **纯 HTML + CSS + JavaScript**（无构建也可部署） |
| 数据 | `content.json` 存题目与文案，改字不用改 HTML |
| 存储进度 | `localStorage` 记录关卡（刷新可续玩） |
| 音效 | `<audio>` + 用户点击后 `play()` |
| 部署 | `git push` 到 `main`，GitHub Pages 源选该分支 |

**不引入 React/Vue 的原因：** 仓库小、你改文案方便、Pages 开箱即用。

**可选增强（以后再加）：**

- 用 GitHub Actions 在 push 时压缩图片。
- 终章生成「在一起天数」用一行 JS 算即可。

## 七、隐私与安全

1. **照片与情书** 只放在密码哈希目录内，不要提交到公开路径。
2. **密码** 用长句 + 数字（如纪念日组合），防暴力猜解。
3. **robots**：页面加 `<meta name="robots" content="noindex, nofollow">`（你现有密码页已有）。
4. **不要在题目里写** 银行卡、身份证、家庭住址等敏感信息。
5. 若仓库 Public，接受「链接+密码」级别的隐私，而非绝对保密。

## 九、交付方式建议（怎么给她）

不要只发 URL。可以：

1. **实体卡**：小卡片写「打开电脑，去这个地址，密码是___」。
2. **二维码**：打印二维码贴在小礼物盒里，扫开后仍是密码页。
3. **分步解锁**：先送花，晚上再发链接，形成二次惊喜。
